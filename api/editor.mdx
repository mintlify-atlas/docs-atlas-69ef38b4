---

The `Editor` class is the central object of every GrapesJS application. It is returned by `grapesjs.init()` and provides access to every sub-module, exposes methods for reading and writing canvas content, manages selection state, and implements a typed event bus. Everything from loading project data to running built-in commands passes through this object.

```typescript
import grapesjs from 'grapesjs';

const editor = grapesjs.init({
  container: '#gjs',
  height: '100%',
  width: 'auto',
});
```

## Methods

### getConfig

Returns the editor configuration object, or the value of a single key when `prop` is provided.

```typescript
getConfig<P extends EditorConfigKeys | undefined = undefined>(prop?: P): EditorConfig | EditorConfig[P]
```

### getHtml

Returns the HTML built inside the canvas.

```typescript
getHtml(opts?: {
  component?: Component;
  cleanId?: boolean;
}): string
```

<ParamField path="opts.component" type="Component">
  Return the HTML of a specific component only. Omit to get the full canvas HTML.
</ParamField>

<ParamField path="opts.cleanId" type="boolean" default="false">
  Remove auto-generated IDs that are not semantically meaningful.
</ParamField>

<CodeGroup>

```typescript TypeScript
const html = editor.getHtml();

// Scoped to one component
const html = editor.getHtml({ component: myComp, cleanId: true });
```

```javascript JavaScript
const html = editor.getHtml();
```

</CodeGroup>

---

### getCss

Returns CSS built inside the canvas.

```typescript
getCss(opts?: {
  component?: Component;
  json?: boolean;
  avoidProtected?: boolean;
  onlyMatched?: boolean;
  keepUnusedStyles?: boolean;
  allowEmpty?: boolean;
}): string | CssRule[]
```

<ParamField path="opts.component" type="Component">
  Return only the CSS that belongs to a specific component.
</ParamField>

<ParamField path="opts.json" type="boolean" default="false">
  Return an array of `CssRule` objects instead of a CSS string.
</ParamField>

<ParamField path="opts.avoidProtected" type="boolean" default="false">
  Exclude rules marked as protected.
</ParamField>

<ParamField path="opts.onlyMatched" type="boolean" default="false">
  Return only rules matched by the passed component.
</ParamField>

<ParamField path="opts.keepUnusedStyles" type="boolean" default="false">
  Force-include all defined rules, even unused ones. Useful when canvas output differs from external output.
</ParamField>

<ParamField path="opts.allowEmpty" type="boolean" default="false">
  Include rules with empty style declarations.
</ParamField>

### setComponents

Replace all components in the canvas. This **overrides** existing content.

```typescript
setComponents(components: ComponentAdd, opt?: AddComponentsOption): this
```

<ParamField path="components" type="string | object | object[]" required>
  An HTML string, a component definition object, or an array of component definitions.
</ParamField>

```javascript
editor.setComponents('<div class="cls">New component</div>');

editor.setComponents({
  type: 'text',
  classes: ['cls'],
  content: 'New component',
});
```

<Warning>This method replaces all existing canvas content. To append without replacing, use `addComponents()`.</Warning>

---

### addComponents

```typescript
addComponents(components: ComponentAdd, opts?: AddOptions): Component[]
```

<ParamField path="components" type="string | object | object[]" required>
  An HTML string, a component definition object, or an array of component definitions.
</ParamField>

<ParamField path="opts.avoidUpdateStyle" type="boolean" default="false">
  When the HTML string contains styles, set this to `true` to skip updating already-existing rules.
</ParamField>

```javascript
editor.addComponents('<div class="cls">New component</div>');
```

### store

Persist the project to the configured storage. Resets `getDirtyCount()` to zero.

```typescript
store<T extends StorageOptions>(options?: T): Promise<ProjectData>
```

```javascript
const saved = await editor.store();
```

---

### load

Load project data from the configured storage.

```typescript
load<T extends StorageOptions>(
  options?: T,
  loadOptions?: { clear?: boolean }
): Promise<ProjectData>
```

<ParamField path="loadOptions.clear" type="boolean" default="false">
  Reset the editor state — including the dirty counter and undo history — after loading.
</ParamField>

```javascript
const data = await editor.load();

// Load and clear history
await editor.load({}, { clear: true });
```

### getProjectData

Serialize the current project to a plain JSON object that can be stored and later restored with `loadProjectData`.

```javascript
const json = editor.getProjectData();
```

### loadProjectData

```typescript

```

<ParamField path="data" type="ProjectData" required>

</ParamField>

<ParamField path="options" type="object">
  Custom options forwarded to `project:load` and `project:loaded` event payloads.
</ParamField>


### setCustomRte

Replace the built-in Rich Text Editor with a custom implementation.

```typescript
setCustomRte<T>(obj: CustomRTE & ThisType<T & CustomRTE>): void
```

The `obj` must implement `enable(el, rte)` at minimum; `disable(el, rte)` and `focus(el, rte)` are also supported.

```javascript
editor.setCustomRte({
  enable(el, rte) {
    rte = new MyRTE(el, {});
    return rte;
  },
  disable(el, rte) {
    rte.blur();
  },
  focus(el, rte) {
    rte.focus();
  },
});
```

---

### setCustomParserCss

Replace the default CSS parser with a custom function. Pass `null` to remove a previously set custom parser.

```typescript
setCustomParserCss(parser: CustomParserCss | null): this
```

The parser receives a CSS string and must return an array of rule objects with `selectors` and `style` fields.

```javascript
editor.setCustomParserCss((css) => {
  const result = [];
  // parse css...
  result.push({
    selectors: '.someclass, div .otherclass',
    style: { color: 'red' },
  });
  return result;
});
```

### log

Emit a structured log event through the editor's event bus.

```typescript
log(msg: string, opts?: { ns?: string; level?: string }): this
```

<ParamField path="msg" type="string" required>
  The message to log.
</ParamField>

<ParamField path="opts.ns" type="string" default="''">
  Namespace (e.g. a plugin name). Generates additional scoped events such as `log-my-plugin` and `log-my-plugin:info`.
</ParamField>

<ParamField path="opts.level" type="string" default="'debug'">
  Severity level: `'debug'`, `'info'`, `'warning'`, or `'error'`.
</ParamField>

```javascript
editor.log('Component loaded', { ns: 'my-plugin', level: 'info' });
// Triggers: 'log', 'log:info', 'log-my-plugin', 'log-my-plugin:info'
```

Register a callback that runs once the editor is fully loaded and rendered. If the editor is already ready when this is called, the callback fires immediately.

## Events

All events are emitted on the editor instance. Listen with `editor.on(eventName, callback)`.

### Core editor events

| Event | Payload | Description |
|---|---|---|
| `update` | — | Triggered on any project change (component added/removed, style change, etc.) |
| `undo` | — | Undo executed |
| `redo` | — | Redo executed |
| `load` | `editor` | Editor fully loaded and canvas rendered |
| `project:load` | `{ project, options, initial }` | Project JSON loaded (initial load or via `loadProjectData`) |
| `project:loaded` | `{ project, options, initial }` | Same as `project:load`, but only fires when the project is non-empty |
| `project:get` | `{ project }` | Fired when project data is requested — use to inject custom keys |
| `log` | `msg, opts` | Log message triggered |
| `telemetry:init` | — | Initial telemetry payload sent |
| `destroy` | — | Editor started tearing down |
| `destroyed` | — | Editor fully destroyed |
| `canvasScroll` | — | Canvas element scrolled |

### Log sub-events

When `editor.log()` is called with a namespace and/or level, additional events fire:

```javascript
// editor.log('msg', { ns: 'plugin-x', level: 'info' })
editor.on('log', (msg, opts) => { ... });
editor.on('log:info', (msg, opts) => { ... });
editor.on('log-plugin-x', (msg, opts) => { ... });
editor.on('log-plugin-x:info', (msg, opts) => { ... });
```

### Module events

Each module emits its own scoped events. See the individual module pages for complete event references:

- [Commands events](/api/commands) — `command:run`, `command:stop`, `command:abort:ID`, etc.
- [Keymaps events](/api/keymaps) — `keymap:add`, `keymap:remove`, `keymap:emit`
- Component events — `component:add`, `component:remove`, `component:update`, etc.
- Storage events — `storage:start`, `storage:end`, `storage:error`, etc.
- Style Manager events — `styleManager:update`, etc.
